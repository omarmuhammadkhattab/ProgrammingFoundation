Why Data Structures?

When you have many related values, storing them in individual variables becomes impractical. Data structures let you organize, store, and operate on collections of data efficiently.

Think of it like organizing your belongings — you could scatter everything on the floor, or you could use boxes, folders, and labeled drawers. Data structures are those containers.

Lists

A list is like an array — an ordered, mutable collection that can hold any mix of data types. It's one of the most versatile tools in Python.

list operations
fruits = ["apple", "banana", "cherry"] # Access by index print(fruits[0]) # "apple" print(fruits[-1]) # "cherry" # Modify (lists are mutable!) fruits[1] = "mango" print(fruits) # ["apple", "mango", "cherry"] # Add & remove fruits.append("kiwi") # add to end fruits.insert(1, "pear") # add at index 1 fruits.remove("apple") # remove by value print(len(fruits)) # length print(fruits.index("kiwi")) # find index fruits.sort() # sort in place
🔒

Tuples

Tuples are like lists, but immutable — once created, the values cannot be changed. Use them when your data should stay constant (like coordinates, RGB colors, or days of the week).

💡 When to use a tuple? If the contents of a collection shouldn't change, use a tuple. It prevents accidental modification and can be slightly faster than a list.
tuple examples
coords = (30.0444, 31.2357) # Cairo coordinates rgb = (255, 212, 59) # Python yellow print(coords[0]) # 30.0444 print(len(rgb)) # 3 # Unpacking — very useful! lat, lon = coords print(f"Lat: {lat}, Lon: {lon}") # This would cause an ERROR (tuples are immutable): # coords[0] = 0 → TypeError!
🎯

Sets

Sets are unordered collections with no duplicate elements. They're great for removing duplicates or checking membership quickly. Inspired by mathematical sets.

set examples
nums = {1, 2, 3, 2, 1, 4} print(nums) # {1, 2, 3, 4} — duplicates removed! # Remove duplicates from a list my_list = [1, 2, 2, 3, 3, 3] unique = set(my_list) print(unique) # {1, 2, 3} # Set operations a = {1, 2, 3} b = {3, 4, 5} print(a | b) # union: {1, 2, 3, 4, 5} print(a & b) # intersection: {3} print(a - b) # difference: {1, 2}
📖

Dictionaries

Dictionaries store data as key-value pairs, like a real dictionary where each word (key) has a definition (value). Keys must be unique and immutable; values can be anything.

dictionary operations
student = { "name": "Ahmed", "age": 22, "grade": "A" } # Access by key print(student["name"]) # "Ahmed" print(student.get("age")) # 22 # Add or update student["city"] = "Cairo" student["age"] = 23 # Iterate for key, val in student.items(): print(f"{key}: {val}") print(len(student)) # 4 (number of keys)
⚖️
