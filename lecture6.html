Functions in Python

Stop repeating yourself — wrap reusable logic into functions. Learn to define, call, and return values from your own functions, and understand how parameters and scope work.

def parameters return default values scope function call
🧩

Why Do We Need Functions?

Functions are a fundamental building block in Python programming. They let you organize code into reusable blocks, making programs more efficient and easier to understand.

Reusability
Write the logic once, use it anywhere. No more copy-pasting the same block of code throughout your program.
Organization
Break complex programs into smaller, manageable modules. Each function does one thing well.
Readability
A well-named function like calculate_average() is self-documenting — the name tells you exactly what it does.
Abstraction
Hide the complexity. You don't need to know how print() works internally — you just call it.
🍞 The toaster analogy: A function is like a toaster — you don't need a different toaster for bread vs. bagels. You just pass in what you want toasted (input / argument), and it gives back something toasted (output / return value). Same toaster, different inputs.
🔬

Anatomy of a Python Function

Every function has the same structure. Here's how to read and write one:

def function_name(param1, param2):
def keyword starts the definition; choose a descriptive name
# function body
Indented block — the code that runs when called
result = param1 + param2
Uses the parameters passed in as input
return result
return hands a value back to the caller
PartWhat it isRequired?
defKeyword that signals a function definition✓ Yes
nameDescriptive identifier; use snake_case✓ Yes
parametersVariables that receive input values (the arguments)Optional
bodyIndented code block that performs the task✓ Yes
returnSends a value back to whoever called the functionOptional
🔑 Parameters vs. Arguments: A parameter is the variable in the function definition (def greet(name)). An argument is the actual value you pass when calling it (greet("Ahmed")).
📞

Defining & Calling a Function

Defining a function creates it — but does not run it. Calling a function is what actually executes its code. You can call the same function as many times as you like with different inputs.

defining vs. calling
# DEFINITION — the blueprint (runs nothing yet) def average(a, b): total = a + b return total / 2 # CALLING — actually runs the function mid = average(10.6, 7.2) # ← function call print(mid) # 8.9 # Call again with different arguments print(average(4, 8)) # 6.0 print(average(100, 200)) # 150.0
↩️

Return Values

The return statement ends the function and sends a value back to the caller. A function without a return statement implicitly returns None. You can also return multiple values at once — Python packs them into a tuple.

return — single, none, multiple
# Single return value def square(n): return n ** 2 print(square(5)) # 25 # No return — implicitly returns None def greet(name): print(f"Hello, {name}!") # side-effect only result = greet("Salma") print(result) # Hello, Salma! then None # Multiple return values (packed as a tuple) def min_max(numbers): return min(numbers), max(numbers) lo, hi = min_max([3, 1, 9, 4, 7]) print(lo, hi) # 1 9 # return exits the function immediately def first_positive(nums): for n in nums: if n > 0: return n # stops as soon as one is found return None # if none found
⚠️ Code after return is unreachable. Once Python hits return, it exits the function immediately. Any statements written after it in the same branch will never run.
⚙️

Default Parameter Values

You can give parameters a default value by writing param=value in the definition. Parameters with defaults become optional when the function is called — if you don't pass them, Python uses the default.

default parameter values
def fun(a, b=3, c="DS502"): return a + b fun(2, 5, "DS503") # a=2, b=5, c="DS503" → 7 fun(4, 6) # a=4, b=6, c="DS502" → 10 fun(2) # a=2, b=3, c="DS502" → 5 # Real-world example: power function def power(base, exponent=2): return base ** exponent print(power(3)) # 9 (squares by default) print(power(3, 3)) # 27 (cubes when specified)
📌 Rule: Parameters with default values must always come after parameters without defaults. def f(a=1, b) is a SyntaxError. Write it as def f(b, a=1).
🔭

Variable Scope

Scope determines where a variable can be seen and used. Variables created inside a function are local — they exist only while the function runs and are invisible from outside.

local vs. global scope
# Global variable — visible everywhere course = "DS502" def show_info(): # Local variable — only exists inside this function student = "Ahmed" print(course) # ✓ can read global print(student) # ✓ can read local show_info() print(course) # ✓ DS502 print(student) # ✗ NameError — student is local!
✅ Best practice: Avoid modifying global variables from inside a function. Instead, pass values in as parameters and get results back via return. This keeps functions self-contained and easier to test.
📚

Examples

example 1 — greet a person by name
def greet(name): print(f"Hello, {name}!") greet("Ahmed") # Hello, Ahmed! greet("Salma") # Hello, Salma!
example 2 — sum of squares of two values
def sum_of_squares(a, b): return a**2 + b**2 print(sum_of_squares(3, 4)) # 9 + 16 = 25 print(sum_of_squares(5, 12)) # 25 + 144 = 169
example 3 — sum of even numbers in a list
def sum_evens(numbers): total = 0 for n in numbers: if n % 2 == 0: total += n return total nums = [1, 2, 3, 4, 5, 6] print(sum_evens(nums)) # 2+4+6 = 12
